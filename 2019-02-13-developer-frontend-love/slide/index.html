Do we even need JavaScript bundlers?

Problems with JavaScript bundlers

There is a build step.

  • Might be slow
  • Adds additional complexity
Problems with JavaScript bundlers

You have to configure it.

  • For special dialects like JSX or CSS modules
  • #0CJS is a good start...
  • ...but every reasonable-sized app will eventually require custom configuration
Problems with JavaScript bundlers

Additional indirection.

  • The code you write is not the code that is executed
  • Configuring source maps can be error prone
  • Source maps are slow to generate

You suddenly have to deal with problems you didn't have before.

The purpose of JavaScript bundlers.

                    

                    

                        
                        
                    

                    

                        
Dev experience

                        

                            They enable JavaScript modules in all browsers.
                        

                        
                        
                    

                    

                        
Dev experience

                        
HTML, CSS and WebAssembly modules.

                    

                    

                        
Dev experience

                        

                            They allow us to use custom languages.
                        

                        
    
                            
  • JSX
    • 
                            
  • Vue Single-File Components
    • 
                            
  • TypeScript
    • 
                        

                    

                    

                        
Dev experience

                        
Short feedback loop.

                        
    
                            
  • 
                                Fast rebuilds in watch mode
                            
    • 
                            
  • Hot module replacement
    • 
                            
                        

                    

                    

                        
                        
                    

                    

                        
Compatibility

                        
They unify module systems.

                        

                            Like ESM, CJS, even WASM and mabe even still AMD.
                        

                    

                    

                        

                            They take care of...
                        

                        
    
                            
  • 
                                ...wiring all imports and exports together
                            
    • 
                            
  • 
                                ...initializing the module graph
                            
    • 
                        

                        

                            You don't need to worry about what module system
                            your dependencies are using.
                        

                    

                    

                        
Compatibility

                        
They unify the underlying platform.

                        

                            Historically, we tried to simulate a CJS environment
                            with Node.js core modules.
                        

                    

                    

                        
Compatibility

                        
They compile down new language features.

                        

                            Via build tools like Babel and PostCSS.
                        

                    

                    

                        
                        
                    

                    

                        
Optimization

                        
They optimize resource loading.

                        
    
                            
  • 
                                By resolving module ids on build time.
                            
    • 
                            
  • 
                                By merging modules that depend on each other
                                into a single chunk.
                            
    • 
                            
  • 
                                They add resource hints for the browser.
                            
    • 
                        

                    

                    

                        
Optimization

                        

                            They allow us to do effective long-term caching.
                        

                        

                            By calculating asset hashes and automatically
                            updating all URLs.
                        

                    

                    

                        
Optimization

                        
They reduce asset file sizes.

                        

                            By removing...
                        

                        
    
                            
  • 
                                unused code (a.k.a. tree-shaking and dead code
                                elimination)
                            
    • 
                            
  • 
                                redundant characters (minification)
                            
    • 
                            
  • 
                                unnecessary bytes (compression)
                            
    • 
                        

                    

                    

                        

                            Well...
                        

                        

                            ...they don't actually do the latter, but they
                            orchestrate other tools which do that.

                            Optimized resource loading.
                        

                    

                    

                        

                            Should we even create chunks with ESMs and H2 push?
                        

                        

                            Push is a new HTTP feature that allows the server to push resources pro-actively to the client.
                        

                    

                    

                        There is no clear answer (yet)
                        
                            and many factors need to be considered.
                        
                    

                    

                        

                            Current problems with ESM and H2 push:
                        

                        
    
                            
  • 
                                Browsers haven't optimized native module initialization yet.
                            
    • 
                            
  • 
                                Inconsistent H2 push implementations.
                            
    • 
                            
  • 
                                Pushing resources that the client has already cached is worse than without push.
                            
    • 
                            
  • 
                                Text compression algorithms (like gzip) are more effective on bigger files.
                            
    • 
                        

                    

                    

                        

                            H2 push is still hard.
                        

                        

                            Rules of thumb for H2 push.
                        

                    

                    

                        

                            As implementations mature, we need to re-evaluate the question whether we should create chunks or not.
                        

                    

                    

                        

                            Let's assume that we really, really wanted to use H2 push.
                        

                        

                            How would it work?
                        

                    

                    

                        

                            There are two possible solutions:
                        

                        
  • 
                            Use an empiric approach like node-h2-auto-push which tries to learn "what to push" by observing client requests or...
                        
    • 
                        
  • 
                            ...use a bundler which generates a dependency graph on build time
                            and pass the result to the H2 server.
                        
    • 
                    
    
                    
    
                        
    
                            H2 push is not the only performance optimization a bundler can do.
                        
    
                    
    
                    
    
                        
    
                            But the problem is: Getting the best performance requires a lot of configuration.
                        
    
                    
    
                    
    
                        
    
                            The solution: Give more data to the bundler so that it can perform automated decisions.
                        
    
                    
    
                    
    
                        
    Data source #1
    
                        
    
                            Static analysis.
                        
    
                    
    
                    
    
                        
    
                            Example: webpackPrefetch: true
                        
    
                        
    
                        
                            Since webpack 4.6.0.
                        
                    
    
                    
    
                        
    
                            Another idea: webpackPreload: true for render critical fonts.
                        
    
                        
    
                        
                            Does not work for render critical fonts yet.
                        
                    
    
                    
    
                        
    
                            It could also add resource hints automatically, like
                            preconnect for all hosts found
                            in the source code.
                        
    
                        
    
                    
    
                    
    
                        
    
                            But this requires a lot manual configuration...
                        
    
                        
    
                            ...which can get outdated over time.
                        
    
                    
    
                    
    
                        
    Data source #2
    
                        
    
                            Automated critical render path tests.
                        
    
                    
    
                    
    
                        
    
                            The idea:
                        
    
                        
      
                            
    • 
                                There is an automated test that navigates to popular routes.
                            
      • 
                            
    • 
                                The bundler captures relevant performance data.
                            
      • 
                            
    • 
                                The bundler decides which resources to optimize and to prioritize.
                            
      • 
                        
    
                    
    
                    
    
                        
    
                            Example: Create optimized subsets of fonts based on the actual usage of glyphs.
                        
    
                        
      
                            
    • 
                                subfont
                            
      • 
                            
    • 
                                glyphhanger
                            
      • 
                        
    
                    
    
                    
    
                        
    
                            Another example: Inline render critical CSS.
                        
    
                        
      
                            
    • 
                                This is already a popular feature of CSS-in-JS libraries.
                            
      • 
                            
    • 
                                critical
                            
      • 
                        
    
                    
    
                    
    
                        
    
                            Running automated critical render path tests plays especially well with the JAMstack.
                        
    
                    
    
                    
    
                        
    
                            But these tests are also error prone because we have to select popular routes.
                        
    
                        
    
                            Instead of artifical usage data, what if we used...
